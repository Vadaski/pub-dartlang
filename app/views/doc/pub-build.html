$ pub build

Use pub build when you’re ready to deploy your web app. When you run pub build, it generates the assets for the current package and all of its dependencies, putting them into a new directory named build.

To use pub build, just run it in your package’s root directory. For example:

$ cd ~/dart/helloworld
$ pub build
Building helloworld......
Built 5 files!

If the build directory already exists, pub build deletes it and then creates it again.

To generate assets, pub build uses transformers. Any source assets that aren’t transformed are copied, as is, into the build directory or one of its subdirectories.

See Assets and Transformers for information on:

  • Where in your package to put assets.
  • What URLs to use when referring to assets.
  • How to use pubspec.yaml to specify which transformers run, and in what order.

Also see pub serve. With pub serve, you can run a development server that continuously generates and serves assets.
